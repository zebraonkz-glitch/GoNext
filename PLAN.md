Документ описывает порядок действий по реализации приложения на основании [PROJECT.md](./PROJECT.md).

Уже выполнено:

## Этап 1. Архитектура и инфраструктура

**Цель:** заложить структуру проекта, типы данных и слой хранения.

### 1.1. Структура каталогов

```
app/                  — экраны (Expo Router)
components/           — переиспользуемые UI-компоненты
db/                   — SQLite: схема, миграции, запросы
services/             — бизнес-логика (места, поездки, файлы, карты)
types/                — TypeScript-типы сущностей
hooks/                — хуки доступа к данным
constants/            — константы приложения
```

### 1.2. TypeScript-типы

Описать интерфейсы по PROJECT.md:

| Сущность    | Ключевые поля |
|-------------|---------------|
| `Place`     | id, name, description, visitlater, liked, dd (lat/lng), createdAt |
| `Trip`      | id, title, description, startDate, endDate, current, createdAt |
| `TripPlace` | placeId, order, visited, visitDate, notes |
| `Photo`     | id, entityType, entityId, filePath, createdAt |

Отдельно зафиксировать формат `dd` (десятичные градусы: `{ latitude, longitude }`).

### 1.3. Локальная база данных (SQLite)

1. Установить `expo-sqlite`
2. Создать файл инициализации БД (`db/database.ts`)
3. Описать схему таблиц:

```
places
trips
trip_places
photos
```

4. Реализовать миграции (версионирование схемы)
5. Написать CRUD-функции для каждой сущности

**Связи:**

```
Trip 1 — * TripPlace * — 1 Place
Place 1 — * Photo
TripPlace 1 — * Photo
```

### 1.4. Хранение фотографий

1. Установить `expo-file-system` и `expo-image-picker`
2. Создать сервис `services/photos.ts`:
   - сохранение файла в каталог приложения
   - запись пути в таблицу `photos`
   - удаление файла при удалении записи
3. Определить соглашение об именах файлов и каталогах

### 1.5. Провайдер данных

Создать React-контекст или набор хуков (`usePlaces`, `useTrips`), которые:

- загружают данные из SQLite при старте
- предоставляют методы создания / изменения / удаления
- обновляют UI после изменений

**Результат этапа:** приложение запускается, БД создаётся, типы и слой данных готовы к использованию на экранах.

## Этап 2. Навигация и главный экран

**Цель:** построить каркас навигации согласно разделу 5 PROJECT.md.

### 2.1. Маршруты Expo Router

```
app/
  _layout.tsx              — корневой layout (PaperProvider)
  index.tsx                — главный экран (меню)
  places/
    index.tsx              — список мест
    [id].tsx               — карточка места
    new.tsx                — создание места
  trips/
    index.tsx              — список поездок
    [id].tsx               — детали поездки (маршрут)
    new.tsx                — создание поездки
  next/
    index.tsx              — следующее место
  settings/
    index.tsx              — настройки
```

### 2.2. Главный экран (`app/index.tsx`)

Заменить заготовку на рабочий экран:

- `Appbar.Header` с заголовком **GoNext**
- Вертикальный ряд кнопок (`Button` из Paper):
  - **Места** → `/places`
  - **Поездки** → `/trips`
  - **Следующее место** → `/next`
  - **Настройки** → `/settings`

### 2.3. Общие UI-компоненты

- `ScreenLayout` — обёртка экрана с AppBar и кнопкой «Назад»
- `EmptyState` — заглушка для пустых списков
- `ConfirmDialog` — диалог подтверждения удаления
- `LoadingIndicator` — индикатор загрузки

**Результат этапа:** пользователь может перемещаться между всеми разделами, экраны пока могут быть пустыми.

## Этап 3. Режим «Места»

**Цель:** реализовать раздел 4.1 PROJECT.md.

### 3.1. Список мест (`/places`)

- `FlatList` / `Searchbar` для поиска по названию
- Карточка места: название, флаги `visitlater` / `liked`, миниатюра фото
- FAB «Добавить место»
- Свайп или меню для удаления

### 3.2. Создание и редактирование места (`/places/new`, `/places/[id]`)

Форма с полями:

| Поле          | Компонент Paper        |
|---------------|------------------------|
| Название      | `TextInput`            |
| Описание      | `TextInput` (multiline)|
| Посетить позже| `Switch`               |
| Понравилось   | `Switch`               |
| Координаты    | `TextInput` × 2 или выбор на карте |
| Фотографии    | галерея + `IconButton` добавить |

Кнопки: **Сохранить**, **Удалить** (на экране редактирования).

### 3.3. Геолокация при создании места

1. Установить `expo-location`
2. Кнопка «Моё местоположение» — подставить текущие координаты в поля `dd`
3. Запросить разрешения при первом использовании

### 3.4. Просмотр места на карте

1. Установить `react-native-maps` (или `expo-maps`)
2. Компонент `PlaceMap` — маркер по координатам `dd`
3. Кнопка «Открыть на карте» на карточке места

### 3.5. Фотографии места

- Выбор из галереи или съёмка камерой (`expo-image-picker`)
- Сетка миниатюр на карточке места
- Просмотр фото в полноэкранном режиме
- Удаление отдельного фото

**Результат этапа:** полный CRUD мест с фото, координатами и просмотром на карте.

## Этап 4. Режим «Поездки»

**Цель:** реализовать раздел 4.2 PROJECT.md.

### 4.1. Список поездок (`/trips`)

- Карточки поездок: название, даты, количество мест, прогресс (посещено / всего)
- Визуальное выделение текущей поездки (`current = true`)
- FAB «Создать поездку»

### 4.2. Создание и редактирование поездки (`/trips/new`, `/trips/[id]`)

Форма:

| Поле        | Компонент              |
|-------------|------------------------|
| Название    | `TextInput`            |
| Описание    | `TextInput` (multiline)|
| Дата начала | `DatePicker` / Modal   |
| Дата окончания | `DatePicker` / Modal|
| Текущая     | `Switch` (только одна поездка может быть `current`) |

### 4.3. Маршрут поездки (упорядоченный список мест)

На экране `/trips/[id]`:

- Список `TripPlace` в порядке `order`
- Для каждого пункта: название места, статус `visited`, дата визита
- Кнопка «Добавить место»:
  - выбрать из существующих (`Place`)
  - или создать новое на месте
- Изменение порядка: drag-and-drop (`react-native-draggable-flatlist` или кнопки ↑↓)
- Отметка «Посещено» — переключатель `visited`, автозаполнение `visitDate`

### 4.4. Заметки и фото визита

На каждом `TripPlace`:

- Поле `notes` — текстовые заметки (`TextInput` multiline)
- Фотографии, привязанные к `TripPlace` (отдельно от фото самого `Place`)
- После отметки «Посещено» карточка меняет вид (дневниковая запись)

### 4.5. Режимы просмотра: план и дневник

На экране поездки переключатель или вкладки:

- **План** — все места маршрута, акцент на непосещённых
- **Дневник** — только посещённые места с заметками и фото

**Результат этапа:** пользователь может создать поездку, собрать маршрут, отмечать посещения и вести дневник.

## Этап 5. Режим «Следующее место»

**Цель:** реализовать раздел 4.3 PROJECT.md.

### 5.1. Бизнес-логика

Сервис `services/nextPlace.ts`:

```
1. Найти поездку с current = true
2. Из trip_places выбрать первое с visited = false, отсортированное по order
3. Вернуть связанный Place или null
```

### 5.2. Экран `/next`

Состояния экрана:

| Ситуация                  | Что показать                              |
|---------------------------|-------------------------------------------|
| Нет текущей поездки       | Сообщение + кнопка «Выбрать поездку»      |
| Все места посещены        | «Маршрут завершён»                        |
| Есть следующее место      | Карточка: название, описание, координаты  |

### 5.3. Действия на экране

- **Открыть на карте** — переход к `PlaceMap` с координатами
- **Открыть в навигаторе** — deep link:
  - Android / iOS: `Linking.openURL` с `geo:` или Google Maps URL
- **Отметить посещённым** — быстрая отметка без перехода в поездку

**Результат этапа:** быстрый доступ к следующей точке маршрута с навигацией.

## Этап 8. Полировка и офлайн-устойчивость

**Цель:** убедиться, что приложение соответствует ограничениям раздела 3 PROJECT.md.

### 8.1. Офлайн-поведение

- Все операции работают без сети
- Карты: кэширование региона или fallback на координаты текстом при отсутствии сети
- Навигатор: `Linking` открывает внешнее приложение (работает офлайн, если карты скачаны)

### 8.2. Обработка ошибок

- Нет координат — скрыть кнопки карты / навигатора
- Нет разрешения на геолокацию — предложить ввести координаты вручную
- Ошибки SQLite — `Snackbar` с понятным сообщением

### 8.3. UX-доработки

- Pull-to-refresh на списках
- Подтверждение при удалении
- Валидация форм (обязательное название, корректные даты)
- Пустые состояния с подсказками

### 8.4. Тема и оформление

- Единая тема Paper (`theme.ts`): цвета, шрифты
- Согласованные отступы и стили карточек
- Иконки из `@expo/vector-icons` / Material Community Icons

**Результат этапа:** стабильное офлайн-приложение с предсказуемым поведением.
